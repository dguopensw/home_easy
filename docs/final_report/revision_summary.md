## 2. 이번 보강에서 해결한 문제점

| 문제 | 해결 방법 |
|------|---------|
| 개발동기가 1문단에 불과 | 독립 2장으로 분리, 6개 주제 5문단 이상으로 확장 |
| 요구사항 분석 없음 | FR-13, NFR-6 표 2개 신규 추가 (코드 근거 병기) |
| 기존 서비스 비교 없음 | 4장 신규 추가, 5개 서비스 비교 표 포함 |
| 시스템 아키텍처 Mermaid 없음 | Mermaid 다이어그램 + 레이어별 역할 표 추가 |
| 사용자 흐름이 3장에 짧게 포함 | 7장으로 독립 분리, UC-1/2/3 + 예외 흐름 표 |
| 실험 결과가 "확인 필요"만 | 13장 실험 설계 독립 추가 (TC 10개, 평가 지표 4개, 판정 기준 표) |
| HTML이 카드형 요약 위주 | 18섹션, 본문 + 표 + 시각화 포함, 30~50화면 수준 |
| 하네스 에이전트 품질 기준 부족 | 각 에이전트에 구체적 작성 기준, 필수 확인 항목, 금지사항 추가 |
| SKILL.md가 3단계 간략 구성 | 9단계 워크플로우, 단계별 필수 산출물 및 확인 항목 명시 |
