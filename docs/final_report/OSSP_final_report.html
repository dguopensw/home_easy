다. 목적 설정

온라인 중고거래 시장에서 가구는 크기와 형태가 구매 결정에 직접적인 영향을 미치는 상품임에도 불구하고, 중고거래 플랫폼의 판매글은 여전히 2차원 사진 중심으로 구성된다. 구매자는 사진만으로 가구의 실제 크기를 파악해야 하는 어려움을 겪으며, 특히 대형 가구일수록 공간 적합성 판단이 어렵다.

본 프로젝트는 AI 기반 이미지 전처리 파이프라인을 통해 중고거래 게시글의 가구 사진에서 3D 모델을 자동 생성하고, Unity 가상 공간에서 실제 스케일에 가깝게 배치함으로써 가구 구매 전 공간 적합성 판단을 보조하는 것을 목표로 한다.

2개발동기 및 문제정의

가. 중고 가구 거래에서 크기 판단의 어려움

국내 중고거래 시장에서 가구는 거래량이 많고 평균 거래 금액도 높은 카테고리 중 하나다. 소파, 침대, 책상, 옷장 같은 대형 가구는 구매 후 반품이나 교환이 매우 어렵고, 이사 시 처분도 쉽지 않기 때문에 구매 전 신중한 판단이 필요하다. 그러나 중고거래 플랫폼에서 가구를 볼 때 구매자가 얻을 수 있는 정보는 대부분 2~5장의 사진과 판매자가 적은 간략한 텍스트에 불과하다.

나. 사진 기반 거래의 구조적 한계

중고거래 가구 사진에는 구조적인 한계가 있다. 판매자는 대부분 스마트폰으로 촬영하며 배경이나 조명을 통제하지 않는 경우가 대부분이다. 가구 뒤에 다른 물건이 쌓여 있거나, 가구 위에 물건이 올려져 있거나, 가구의 일부가 프레임 밖으로 잘려나간 사진이 자주 올라온다. 단일 이미지에서는 깊이(depth) 정보가 손실되기 때문에, 배경과 가구의 거리 관계를 파악하기 어렵다.

다. 판매글 치수 정보의 불규칙성과 부재

일부 판매자는 정확한 WxDxH 수치를 제공하지만, 많은 판매글에서 치수가 아예 없거나 "꽤 큰 편", "2인용 소파" 같은 모호한 표현만 사용된다. "120*60*75", "가로120 세로60 높이75", "W1200×D600×H750" 같은 다양한 형식을 모두 인식해야 하며, cm와 mm가 혼용되는 경우도 있다.

라. 공간 적합성 판단 수단의 부족

"너비 150cm짜리 소파"라고 해도 내 거실에 어떻게 놓일지 머릿속으로 그리는 것은 쉽지 않다. 실제 공간에서 테이프를 바닥에 붙여 시뮬레이션하는 방식도 있지만, 이는 번거롭고 높이나 깊이는 파악하기 어렵다. 3D 시각화가 이 문제를 해결할 수 있는 가장 직접적인 방법이지만, 기존 서비스들은 이를 중고 가구 거래에 적용하지 못하고 있다.

마. 기존 서비스의 한계와 우리 접근법

IKEA Place, IKEA Kreativ, 오늘의집 AR 등 기존 3D 가구 배치 서비스는 사전 제작된 3D 자산(해당 브랜드 카탈로그)을 기반으로 작동한다. 이 방식은 카탈로그에 등록된 제품에만 적용 가능하며, 임의의 중고 가구 이미지는 처리할 수 없다. 당근마켓, 중고나라는 거래 중개 플랫폼으로서 3D 시각화 기능 자체를 제공하지 않는다.

본 프로젝트는 이 간극을 메운다. 중고거래 URL만 입력하면 크롤링 → SAM3 세그멘테이션 → 인페인팅 → 치수 추정 → TRELLIS 3D 생성까지 자동으로 수행하여, 등록된 3D 자산이 없는 개별 중고 상품에도 적용 가능한 3D 시각화 흐름을 실현한다.

나. 핵심 차별점

기존 3D 가구 배치 서비스들은 공통적으로 사전 제작된 3D 자산(CAD 모델 또는 포토그래메트리 스캔)을 기반으로 작동한다. 이 방식은 카탈로그에 등록된 제품에만 적용 가능하며, 임의의 중고 가구 이미지는 처리할 수 없다.

본 프로젝트는 근본적으로 다른 접근법을 취한다. 중고거래 URL 하나를 입력하면 크롤링 → SAM3 세그멘테이션 → 인페인팅 → TRELLIS 3D 생성까지 자동 처리된다. 등록된 3D 자산이 존재하지 않는 개별 중고 상품에도 적용 가능한 흐름이 핵심 차별점이다.

9이미지 전처리 (SAM3 + 인페인팅)

핵심 4축 ① — 담당자가 구현 내용을 직접 채워야 하는 섹션입니다.

가. 전처리 파이프라인 개요

3D 생성 품질은 생성 모델 자체뿐 아니라 입력 이미지의 품질에 크게 영향을 받는다. 중고거래 이미지는 배경이 복잡하고 가구 위에 물체가 놓인 경우가 많아, TRELLIS 입력 전 SAM3 중심 전처리 파이프라인으로 이미지를 정제한다.

나. SAM3 세그멘테이션

선택 이유: 중고거래 이미지의 배경이 통제되지 않아 rembg 등 일반 배경 제거 도구로는 정확도가 부족하다. SAM3는 점(point)/박스(box) 프롬프트로 가구 영역을 명확하게 타겟팅할 수 있어, 복잡한 배경에서도 정확한 마스킹이 가능하다.

처리 흐름: 원본 이미지 → SAM3 raw mask → 형태학적 연산 + alpha matting으로 정제 → final mask + 소프트 알파 마스크 → 03_final_cutout.png 생성

SAM3 이중 호출 구조: Stage 6(기본 마스킹) + Stage 14(인페인팅 후 재실행 → 06_generation_cutout 생성)

다. 장애물·오염물 분석 및 인페인팅

분석 방법: GPT-4o Vision을 활용하여 가구 주변 장애물(obstacle: 가구 위에 놓인 물체)과 오염물(contaminant: 3D 생성 방해 배경 요소)을 감지한다.

10치수 측정

핵심 4축 ② — 담당자가 구현 내용을 직접 채워야 하는 섹션입니다.

가. 치수 측정이 필요한 이유

TRELLIS가 생성하는 GLB 파일은 절대적인 실세계 스케일 정보를 포함하지 않는다. Unity에서 GLB를 import하면 임의의 단위 크기로 표시되므로, 실제 공간 배치 시뮬레이션을 위해서는 GLB의 스케일을 실제 가구 치수에 맞게 보정해야 한다. 치수 측정 모듈의 목적은 이 보정값 확보다.

라. GPT Vision 추정 방식과 한계

판매글 치수가 없을 때 DimensionEstimatorService가 02_measurement.png(SAM3 마스크 윤곽선을 원본 위에 합성한 이미지)를 GPT-4o Vision에 입력한다. GPT Vision은 가구 유형별 일반적 크기 범위와 이미지 내 비율을 참고하여 추정값을 생성한다.

⚠️ 한계: 단일 이미지에서는 깊이 정보가 없으므로 추정 신뢰도가 낮다(confidence=low). 사람이나 다른 물체가 함께 찍혀 있지 않으면 가구의 실제 크기 판단이 어렵다.
마. Unity 스케일 보정 결과
⚠️ Unity 스케일 보정 적용 화면 캡처 삽입 필요 스케일 보정 전/후 비교 화면 권장

📋 치수 측정 담당자 작성 가이드

  • 파싱 성공률: 수집한 판매글 샘플에서 치수 파싱이 성공한 비율과 실패 패턴을 기재하세요.
  • GPT Vision 추정 오차: 실제 치수와의 오차를 측정하여 기재하세요 (예: 평균 오차 XX%).
  • result.json dimensions 예시: 실제 파이프라인 실행 결과의 dimensions 필드 예시값을 삽입하세요.
  • 어려운 케이스: 치수 파싱이 실패한 판매글 패턴 예시를 기재하세요.
  • Unity 스케일 보정 결과: 보정 전/후 Unity 화면 비교 캡처를 삽입하세요.

12Unity 구현

핵심 4축 ④ — 담당자가 구현 내용을 직접 채워야 하는 섹션입니다.

가. Unity 구현 목표

TRELLIS가 생성한 GLB 파일을 Unity에 import하여, result.json의 치수 정보를 기반으로 실제 스케일에 가까운 가구 3D 모델을 가상 공간에 배치하는 것이 Unity 구현의 목표다. 구매자가 "이 가구가 내 방에 들어갈 수 있는가"를 3D 공간에서 직접 확인할 수 있도록 한다.

나. Unity GLB 배치 과정
단계작업 내용비고
1. GLB 다운로드result.json의 model_generation.glb_url에서 GLB 파일 다운로드—
2. Unity ImportGLB 파일을 Unity 프로젝트 Assets에 importUnity 2022 이상 권장
3. 스케일 보정result.json dimensions 참조, Transform.Scale 적용 (1 unit = 1 meter)width_cm / 100.0 기준
4. 공간 배치가상 방 모델 위에 가구 3D 배치 및 위치 조정—
5. 결과 캡처배치 완료 화면 캡처 및 보고서 삽입—
다. 스케일 보정 수식
Transform.Scale = (width_cm / 100.0, height_cm / 100.0, depth_cm / 100.0)
예시: 소파 width=165cm, height=80cm, depth=90cm → Scale = (1.65, 0.80, 0.90)
라. Unity 배치 결과 시연
⚠️ Unity 배치 결과 캡처 삽입 필요 스케일 보정 전/후 비교 화면 권장
⚠️ 가구 유형별(소파, 책상, 의자 등) 배치 화면 캡처 삽입 필요

📋 Unity 구현 담당자 작성 가이드

  • Unity 환경 설정: 사용 Unity 버전, GLB 임포터 설정, 가상 방 씬 구성 방법을 기재하세요.
  • 스케일 보정 검증: 실제 가구 치수(줄자)와 Unity 배치 스케일 일치 여부를 확인하고 기재하세요.
  • 다양한 가구 배치 결과: 소파, 의자, 책상 등 유형별 Unity 배치 화면을 캡처하여 삽입하세요.
  • 스케일 불일치 케이스: result.json 치수가 부정확하여 실패한 케이스와 원인을 기재하세요.
  • 자동화 가능성: Unity 스크립트에서 result.json을 자동으로 읽어 스케일 적용하는 방향을 검토하세요.

15기대효과 및 활용방안

가. 사용자 구매 결정 보조

중고 가구 구매자는 3D 모델을 통해 2D 사진으로는 파악하기 어려운 입체적 형태와 크기를 확인할 수 있다. Unity에서 치수 기반 스케일 보정이 적용된 3D 모델은 가구가 실제 공간에서 어떻게 배치될지를 직관적으로 판단하는 데 도움을 준다.

나. 중고거래 신뢰도 개선

치수 정보가 불완전한 판매글에서도 AI 기반 추정으로 대략적인 크기 정보를 제공하여 "생각보다 컸다/작았다" 유형의 문제를 줄이는 데 기여할 수 있다.

다. 기술적 확장 가능성
  • SAM3 전처리 파이프라인: 가구 외 의류, 전자제품 등 다른 중고거래 카테고리에도 적용 가능
  • 크롤링 모듈: 당근마켓, 중고나라 외 추가 플랫폼 확장 가능한 서비스 구조
  • 인테리어 시뮬레이션 서비스와의 연계: 직접 촬영한 기존 가구 사진도 동일 파이프라인 적용 가능
  • Unity 자동화: result.json 기반 스케일 자동 적용 스크립트로 확장 가능
